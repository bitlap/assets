# 1.1.0

### 资产管理改版
- **分组折叠**：资产按现金、定期存款、理财/基金分类展示，支持折叠/展开
- **分类拖拽排序**：分类标题可自由拖拽调整顺序，自动持久化
- **资产拖拽排序**：同一分类内的资产可拖拽调整顺序，跨分类拖拽被阻止并提示
- **键盘适配**：编辑对话框自动避让键盘，输入体验更顺畅
- **货币选择器修复**：分类内货币选择下拉菜单不再被裁剪，支持滚动
- **跨分类保护**：分类标题拖到资产区域提示「请在分类标题之间拖拽」，资产拖到其他分类提示「不能移动到其他分类」

### 资产管理代码重构
- 提取 `AssetConfig` — 所有资产相关文案和常量集中管理
- 提取 `AssetFlatItem` — 列表渲染模型独立为公开类
- 提取 `AssetReorderUtil` — 拖拽排序逻辑独立为纯函数
- 提取 `AssetCalculator.getTotalByType` — 按类型汇总资产价值
- `assets_page.dart` 从 700+ 行精简至纯 UI + 状态管理

### 盈亏曲线
- 新增后台自动记录盈亏快照（每 10 分钟），盈亏曲线数据更完整

### iCloud 同步
- 同步机制改为本地优先 + 时间戳比对，离线也能正常使用，多设备同步更稳定
- 优化同步开关的描述文案

### 设置
- 平仓后默认保留持仓记录和iCloud同步（可在设置中关闭）
- 新增第三方库和数据来源信息
- 默认手续费类型改为固定金额
- 所有配置修改后立即同步 iCloud

### 国际化
- App 语言统一为中文
- 所有文案集中管理，后续维护更方便

### 性能
- 首次刷新等待时间从 30 秒缩短至 10 秒
- 盈亏曲线分粒度存储，减少空间占用

### 手续费
- 买入/卖出操作支持输入手续费（买入增加成本，卖出减少收益）
- 支持按比例或固定金额两种计算方式
- 加仓/建仓对话框根据价格和股数自动计算手续费
- 手续费设置 iCloud 同步

### 持久化
- 移除 shared_preferences 依赖，设置全部改用文件系统 JSON 存储（与股票/资产一致）
- 添加资产排序设置（资产排序列、排序方向）持久化支持

## 修复
- 新建持仓时涨跌幅不再错误显示为当日涨跌幅
- 修复更换设备时，本地空数据覆盖远程iCloud数据
- 修复货币切换后，存储的收益快照的货币不统一
- 修复添加资产弹窗点击选项后因同步 pop/push 导致 _HighlightModeManager 崩溃
- 修复排序状态下首次拖拽排序列表被打乱的问题
- 修复后台 WorkManager 任务更新行情后未重新计算盈亏，导致记录的收益快照值错误
- 修复 _refreshAll 在 setState 之前记录收益快照，使用旧值而非最新盈亏
- 修复从 iCloud 加载股票数据后未重新计算每只股票的盈亏
- 修复切换货币后未持久化默认货币，下次启动丢失
- 修复首次打开 App 未立即获取汇率，需等待首次刷新
- 修复 DraggableFab 从后台返回后位置未限制在屏幕内
- 修复资产/股票列表筛选排序未触发重新加载（无 loading 状态）
- 修复股票页面缺少 loading 加载状态
- 修复 iCloud 同步/保存未 await 导致的竞态问题
